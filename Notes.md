# Заметки (для себя)
```
[Тип символа / Описание]
1) Терминал (terminal) - это конечный символ, который встречается в тексте программы (в исходном коде). Он не раскрывается на другие правила.
2) Нетерминал (nonterminal) - это абстрактное правило, которое описывается через другие нетерминалы и/или терминалы.

>------<
Вариант I:
Нетерминал_1 % Терминал_1;

Вариант II:
Нетерминал_1 % Терминал_1;
Нетерминал_2 % Терминал_2;

Вариант III:
Нетерминал_1 % Терминал_1;
Нетерминал_2 % Нетерминал_1;
>------<

[Пример]
1) Одинарный символ, который может выражать любую цифру: 0-9, букву: A-Z a-z, оператор: * / + - % и другие.
2) Группа символов, которые могут выражать фрагмент числа или слова.

[Аналогия]
1) Как «кирпич» — мельче уже не разобрать.
2) Как «формочка» или «узор», который строится из кирпичей.

[Скелет]
1) Порядковый номер конструкции (идентификатор), `автоинкрементируемый`.
2) Название конструкции (нетерминал).
3) Определение конструкции (нетерминал/терминал), с описанием правил.

<-/ Native /->:

1.0) void CreateConstruction();
~~ 1.1) void SetConstructionIdentifier(short identifier); ~~
1.2) short GetConstructionIdentifier(const char *construction_name);
1.3) void DestroyConstruction();

2.1) void SetConstructionName(const char *construction_name);
2.2) char *GetConstructionName(short identifier);
```
# Описать грамматику таким образом, чтобы она позволяла определять синтаксические конструкции - вида:
```
1) Любую цифру (целочисленную): 0-9.
2) Любую букву (английского и русского алфавита): A-Z, a-z, А-Я, а-я.
3) Знак: *, /, +, -.

4) Любое число (целочисленное): 0-999.
5) Любое слово: см.п.2.
```
